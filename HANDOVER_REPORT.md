# 新闻热点聚合网站 — 移交审查报告

## 一、项目概述

**项目路径**: `E:\WorkBuddy\news_hot_site\`
**运行端口**: `http://localhost:8084/`
**技术栈**: FastAPI + SQLite + Jinja2 + Google Translate Free API

这是一个新闻聚合网站，自动从 30 个 RSS 源（15 国内 + 15 国外）抓取文章，自动翻译英文内容为中文，按热度排序展示。

### 核心功能
- 首页：热点板块 + 时间线文章列表
- 精选页：标记为精选的文章
- 搜索：按标题/摘要搜索
- 日报：按日期查看文章
- API：JSON 接口供外部调用
- 定时任务：每 6 小时自动抓取+翻译+重算热点

## 三、已知问题（请 Codex 重点审查）

### 1. 翻译可靠性
- 使用 Google Translate 免费 API，无 API Key 限制
- 存在翻译失败率（部分长文本可能被截断）
- `asyncio.run()` 在 FastAPI 异步路由中调用需要线程隔离（已修复但需验证）

### 2. 爬虫健壮性
- 30 个信源中部分返回 429/超时（如央视新闻、知乎日报）
- 中文 RSS 源格式不统一，feedparser 可能解析失败
- 缺少失败信源告警机制

### 3. 数据一致性
- `upsert_article` 使用 `COALESCE` 处理翻译字段时有空字符串问题（已改用 CASE WHEN 修复）
- 热点分数计算依赖 `published_at` 时间戳，格式不统一可能导致衰减计算错误

### 4. 前端体验
- 翻译质量无法保证（直译 vs 意译）
- 没有"原文/译文"切换功能
- 文章详情页没有显示翻译来源标记

### 5. 安全性
- 搜索输入有长度限制（200字符），但缺少 XSS 防护（Jinja2 自动转义，但需确认）
- 数据库文件在项目中，无备份机制
- 无速率限制（API 可被频繁调用）

### 6. 部署
- 无 systemd/service 管理，需手动启动
- 无日志轮转，`uvicorn` 日志可能堆积
- 无 HTTPS 配置
